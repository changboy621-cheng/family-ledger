# FamilyLedger 視覺優化設計（方向 B：清爽現代風）

日期：2026-06-19
背景：家人加入後反映介面「不好看」。從實機截圖診斷後，做不破壞功能的視覺優化（不重建、沿用現有架構）。

## 目標
讓夫妻日常使用的記帳 App 看起來乾淨、有質感、手機好讀，並修掉看起來像壞掉的圖表。

## 診斷（截圖實證）
1. 圖表 `overflow-x-auto` 橫向捲動，資料落在月底，畫面停在月初空白段 → 看起來一片平、像壞掉。
2. 卡片白底、頁面底色也接近白 → 扁平、缺層次。
3. 配色冷（靛藍 #4F46E5 ＋ slate 灰）→ 不像「家」。
4. USD 永遠顯示 $0.00（總覽卡 6 行有 3 行是無用零元）。
5. 每張卡掛一句教學式說明 → 囉嗦。
6. 設定頁顯示名稱唯讀，使用者要求可隨時改。

## 範圍與做法

### 1. 配色（方向 B 清爽現代風）
- Tailwind token：`family` #4F46E5 → **#1D9E75（湖綠）**；`personal` → **#378ADD（天藍）**。
- 新增淡色 token：`familySoft` #E1F5EE、`personalSoft` #E6F1FB，供關鍵數字卡底色與選取態用。
- 頁面底色 `page` 調為較有對比的冷灰，白卡更跳。
- 清掉零散硬寫的 `indigo-50/100`、`ring-indigo-100`，改用 token（`bg-familySoft`、`focus:ring-family/30`）。
- 成員頭像綠/紅（avatar_color）維持不變＝代表「人」的身分。

### 2. 圖表（ExpenseTrendChart）
- 移除橫向捲動，長條 `flex-1` 撐滿寬度；縮短圖高去除大片留白。
- 長條最小高度確保可見；湖綠填色。
- 日標籤過密時稀疏顯示（每 5 天＋最後一天）。
- 無資料維持友善空狀態。

### 3. 精簡資訊
- 幣別零元不顯示：USD 全部為 0 時整段隱藏（LedgerPage 總覽、DualCurrencyDisplay）。
- 移除各分析卡的教學式副標題（分類/趨勢/支出人/前三大）。

### 4. 卡片層次
- 卡片統一 `rounded-xl`、邊框與底色拉開對比；關鍵數字（本月支出/結餘）用淡色底 hero。

### 5. 設定頁：顯示名稱可隨時改
- 唯讀改為可編輯（點擊→輸入→存檔），寫入既有 `user_profiles.display_name`。
- 純驗證邏輯（去空白、非空、長度上限）抽 lib 寫單元測試；Supabase 寫入放 useAuth。
- 成功/失敗 Toast 回饋。

## 不做（YAGNI）
- 不改資料結構（改名只寫既有欄位）。
- 不動 P2 交易操作體驗、P6-P8（之後另議）。

## 驗證
- 每步 `npm run build` / `lint` / `test` 全綠。
- 公開頁（登入/加入）以本地預覽截圖驗證新配色；登入後頁面以 DEV 暫時預覽路由渲染純展示元件截圖驗證圖表/卡片，驗證後移除。
- 改名功能寫測試。
- 全部完成、使用者看過 OK 才 push 上線（push 觸發 Vercel 自動部署）。
